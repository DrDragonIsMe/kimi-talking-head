# Agent Guidelines

## Project

`kimi-talking-head` generates 1080×1920 portrait talking-head videos from articles. Pipeline: article → script → TTS → Whisper → subtitles → storyboard → scene visuals → InfiniteTalk lip-sync → Remotion render.

## Key Files

- `scripts/pipeline.sh` — main orchestration, resumable via `temp/<run>/.pipeline_state.json`. Pre-flight runs `scripts/lib/validate_config.sh` (aborts on invalid `content_overlay.subtitles.dna`) and, before the `script` phase, `scripts/validate_article.js` (article quality; warn-only unless `STRICT_ARTICLE_CHECK=1`). Host profile resolution: 3rd positional arg > `HOST_PROFILE` env (absolute path, injected by the web admin for multi-host) > `config/host_profile.json`. `video_layout.aspect` maps to the render composition (`9:16`→`TalkingHeadVideo`, `16:9`→`TalkingHeadVideoLandscape`, `1:1`→`TalkingHeadVideoSquare`); `video_layout.preview.enabled=true` renders a 0.33× low-res `temp/<run>/preview.mp4` before the final render.
- `scripts/render_with_reused_media.sh` — reuse existing audio/lip video for style/title-only changes.
- `scripts/lib/subtitle_segmentation.js` (+ sibling `.d.ts`) — single source of truth for subtitle cue segmentation (`findSplitIndex` unified on the enhanced version: break-budget check + dangling punctuation merges ≤2). Consumed via `require` by `scripts/parse_srt.js` and via `import` by `src/hooks/useSubtitles.ts` and `src/utils/keywordMatcher.ts`.
- `scripts/validate_article.js` — article quality pre-check CLI (exit 0/1 + stdout JSON `{ok, checks}`): 100–10000 chars, code-block share <30%, table rows <10, Chinese share ≥50%.
- `scripts/lib/remote_job.sh` — shared SSH/SCP options, conda env activation resolution, nohup remote job submit, and status polling (timeout + ssh-failure circuit breaker) used by `tts_index.sh` / `infinitetalk.sh` / `musetalk.sh`. `remote_job_select_worker` picks a GPU worker from optional `workers[]` in `config/servers.json` (round-robin via `REMOTE_JOB_RR_STATE` cursor + `ConnectTimeout=5` SSH precheck, falls back to primary when unset/all down).
- `scripts/align_subtitles.py` — word-level Whisper JSON + script character-level alignment; also emits `subtitles_words.json` (karaoke word timings).
- `scripts/locate_hero_moments.js` — locates LLM-picked `hero_phrase` (storyboard) in word timings → `hero_moments.json`.
- `src/themes/captions/` — caption DNA registry (`classic` / `loud` / `keynote` / `cream` / `editorial` / `documentary`); selected via `content_overlay.subtitles.dna`. Hero keyword pops are rendered by `src/components/HeroOverlay.tsx` under every DNA.
- `src/index.tsx` — three compositions share one root: `TalkingHeadVideo` (1080×1920 portrait, visual-regression baselines), `TalkingHeadVideoLandscape` (1920×1080), and `TalkingHeadVideoSquare` (1080×1080); `video_layout.aspect` (`9:16`/`16:9`/`1:1`) selects aspect branches in `PortraitHybridLayout` / `TitleCard`. `scripts/pipeline.sh` picks the composition from profile `video_layout.aspect`; `scripts/test_compositions.js` guards the registration in `npm test`.
- `src/utils/sceneMotion.ts` — deterministic Ken Burns + crossfade math for scene visuals.
- `scripts/build_scene_visuals_from_existing.js` — scene assets cached globally by query/prompt sha1 in `public/scene_visuals/_cache/` (`SCENE_VISUALS_CACHE_DIR` override), symlink on hit, LRU cap 500 files / 2GB (cache dir gitignored).
- `config/host_profile.json` — host photo, voice reference, template, layout preset, brand copy.
- `config/servers.json` — SSH + GPU server paths, updated by `scripts/detect_paths.sh`.
- `api/server.js` + `api/public/` — web admin (Express + no-build static SPA): job CRUD, run/rebuild/retry/stop/clone, script read/write (+ automatic `script.v{N}.txt` backups and `GET .../script/versions` / `?version=N`), preview, grouped config form. Job state in `api/jobs/<uuid>/state.json`; child processes tracked in a Map, stop kills the process group. `PIPELINE_SCRIPT` env hook overrides the pipeline script path (used by tests). All run paths (full/rebuild/retry) invoke the same pipeline script; rebuild is just a version with invalidation forced to `render`; retry re-runs the same version (optional `{phase}` injects `FORCE_<PHASE>=1`). Optional `WEB_TOKENS="user:token,…"` enables per-user auth (owner-scoped 404, per-user `api/templates.<user>.json`; `?access_token=` fallback for media/EventSource). Jobs may carry `webhookUrl` — fired (3 attempts) when a version reaches a terminal status; delivery state persists on the version record as `webhookDelivery {status,attempts,lastAttemptAt,lastError}` and pending deliveries resume at startup without duplicates. Multi-host: `POST /api/v1/jobs` accepts `hostProfile` (a `.json` file under `config/hosts/`), injected as `HOST_PROFILE`. New-job article pre-check (400 by default; `ARTICLE_VALIDATE_MODE=warn`, `ARTICLE_VALIDATE_SCRIPT`). Cron schedules via node-cron (`POST/DELETE /api/v1/jobs/:id/schedule`, restored at startup) and unauthenticated external trigger `POST /api/v1/trigger/:token` (per-job `triggerToken` is the credential, route registered before the auth middleware). Progressive preview: watcher detects `temp/<run>/preview.mp4` → SSE `preview_ready` + `GET /api/v1/jobs/:id/preview` stream (Range). `listJobs`/`getJob` use an mtime-invalidated in-memory cache (`api/job-store.js`); `readPipelineState` is cached by `(mtimeMs,size)` fingerprint. API surface documented in `openapi.yaml` (v2.3.0).
- `api/versioning.js` — versioned runs (each run/rebuild appends a version: v1=outputName, vN=`<outputName>_vN`), config-hash snapshots (`configDirty`), phase-level reuse (`computeInvalidationPhase` + `prepareReuseWorkdir`: clone previous workdir with `fs.cpSync`, reset the invalidated phase and everything after it to pending so pipeline.sh resume skips unchanged phases), duration aggregation for `GET /api/v1/estimates` (`aggregateEstimates`), and `estimateCost(articleText, configOverrides)` → `{tokens, seconds}` served as `costEstimate` on `GET /api/v1/jobs/:id`.
- `api/events.js` — SSE broadcaster behind `GET /api/v1/events` (subscribe/publish/closeAll; job status transitions are published from server.js call sites; keepalive comment frames every `SSE_KEEPALIVE_MS`, default 25000). `api/templates.json` — named config-template store (user data, gitignored; seeded with 「知识科普」/「发布会风」 on first read).
- `server/` — GPU server deployment scripts and model checklist.

## Conventions

- Use `bash scripts/pipeline.sh <article> <run>` for full runs.
- Use `FORCE_<PHASE>=1` to re-run a specific phase.
- Run `npm test` before committing; `npm run test:fast` skips the API integration suite and `tsc` for a quicker loop. Use `npm run test:visual` for render-layer changes (re-bless baselines with `UPDATE_BASELINE=1` only after eyeballing the new render).
- `node-cron` is a runtime dependency (job schedules); don't remove it from `package.json`.
- Keep generated media (`temp/`, `output/`, `public/audio.wav`, `public/host_video.mp4`, etc.) out of git.
- Update `README.md`, `RELIABILITY.md`, `.kimi/kimi_pipeline.md`, and `server/*.md` when changing architecture or deployment.
- Never commit `config/host_profile.json`, `config/servers.json`, or `.env`.
