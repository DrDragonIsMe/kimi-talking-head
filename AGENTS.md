# Agent Guidelines

## Project

`kimi-talking-head` generates 1080√ó1920 portrait talking-head videos from articles. Pipeline: article ‚Üí script ‚Üí TTS ‚Üí Whisper ‚Üí subtitles ‚Üí storyboard ‚Üí scene visuals ‚Üí InfiniteTalk lip-sync ‚Üí Remotion render.

## Conventions

- Use `bash scripts/pipeline.sh <article> <run>` for full runs.
- Use `FORCE_<PHASE>=1` to re-run a specific phase.
- Run `npm test` before committing; `npm run test:fast` skips the API integration suite and `tsc` for a quicker loop. Use `npm run test:visual` for render-layer changes (re-bless baselines with `UPDATE_BASELINE=1` only after eyeballing the new render).
- `node-cron` is a runtime dependency (job schedules); don't remove it from `package.json`.
- Keep generated media (`temp/`, `output/`, `public/audio.wav`, `public/host_video.mp4`, etc.) out of git.
- Update `README.md`, `RELIABILITY.md`, `.kimi/kimi_pipeline.md`, and `server/*.md` when changing architecture or deployment.
- Never commit `config/host_profile.json`, `config/servers.json`, or `.env`.
